## 1. 字符串混淆

### 原理

将代码中的字符串常量进行加密或编码，运行时通过解密函数还原。

### 常见实现

#### 1.1 字符串拼接混淆
```javascript
// 原始
var password = "secret123";

// 混淆后
var _0x1234 = "sec" + "ret" + "12" + "3";
```

#### 1.2 Unicode 转义混淆
```javascript
// 原始
console.log("Hello");

// 混淆后
console.log("\x48\x65\x6c\x6c\x6f");
```

#### 1.3 Base64 编码混淆
```javascript
// 原始
var apiKey = "sk-12345678";

// 混淆后
var _0x = atob("c2stMTIzNDU2Nzg=");
```

### 反混淆策略

1. **静态分析**: 识别解密函数，通过模拟执行获取原始字符串
2. **动态分析**: Hook 解密函数，运行时获取明文
3. **AST 重构**: 识别字符串拼接模式，静态拼接还原

## 2. 变量名混淆

### 原理

将具有语义意义的变量名替换为无意义的短名称（如 a, b, _0x1234）。

### 示例

```javascript
// 原始
function calculateSum(numbers) {
    let sum = 0;
    for (let i = 0; i < numbers.length; i++) {
        sum += numbers[i];
    }
    return sum;
}

// 混淆后
function _0x1a2b($0x3c4d, $0x5e6f) {

        $0x7g8h += $0x3c4d[$0x9i0j];
    }
    return $0x7g8h;
}
```

### 反混淆策略

1. **基于作用域分析**: 根据变量的使用范围和调用关系推断语义
2. **基于模式匹配**: 识别常见模式的特征（如数组遍历、DOM 操作）
3. **基于类型推断**: 根据变量使用方式推断其预期类型

## 3. 控制流平坦化

### 原理

将代码的执行顺序打乱，用状态变量和 switch/while 语句控制执行流程。

### 原始代码

```javascript
function foo(x) {
    if (x > 0) {
        console.log("positive");
    } else {
        console.log("non-positive");
    }
}
```

### 平坦化后

```javascript
function foo(_0x) {
    var _0x1a2b = 0;
    while (true) {
        switch (_0x1a2b) {
            case 0:
                if (_0x > 0) {
                    _0x1a2b = 1;
                } else {
                    _0x1a2b = 2;
                }
                break;
            case 1:
                console['log']("positive");
                _0x1a2b = 3;
                break;
            case 2:
                console['log']("non-positive");
                _0x1a2b = 3;
                break;
            case 3:
                return;
        }
    }
}
```

### 反混淆策略

1. **状态机分析**: 解析 switch 结构，构建状态转换图
2. **拓扑排序**: 根据状态转换关系还原原始执行顺序
3. **死代码消除**: 移除状态变量和分发器代码

## 7. JSVMP 虚拟机保护

### 原理

将 JavaScript 代码编译为自定义字节码，通过虚拟机解释执行。

### 核心特征

1. **字节码数组**: 包含指令序列的常量数组
2. **分发器**: switch 或 while 结构实现指令分发
3. **操作数栈**: 模拟栈帧操作
4. **寄存器/上下文**: 存储虚拟机状态

### 示例结构

```javascript
function vm_protected() {
    // 字节码
    var bytecode = [0x01, 0x02, 0x03, 0x04, ...];
    
    // 寄存器
    var pc = 0;      // 程序计数器
    var stack = [];  // 操作数栈
    var ctx = {};    // 上下文
    
    // 分发器循环
    while (true) {
        switch (bytecode[pc++]) {
            case 0x01: // LOAD
                stack.push(bytecode[pc++]);
                break;
            case 0x02: // ADD
                var b = stack.pop();
                var a = stack.pop();
                stack.push(a + b);
                break;
            // ... 更多指令
            case 0xFF: // HALT
                return stack.pop();
        }
    }
}
```

### 反混淆策略

1. **字节码提取**: 识别字节码数组和常量
2. **指令模式识别**: 分析分发器识别指令类型
3. **语义还原**: 将字节码翻译为等价的 JavaScript
4. **模拟执行**: 运行时执行字节码获取结果

## 8. 其他混淆技术

### 8.1 作用域混淆

```javascript
// 使用 with 语句混淆作用域
with ({}) {
    console.log("test");
}
```

### 8.2 正则表达式混淆

```javascript
var pattern = /(?=a)b(?=c)d/;
```

### 8.3 十六进制字符串混淆

```javascript
var str = "\x48\x65\x6c\x6c\x6f";
```

### 8.4 数字常量混淆

```javascript
// 使用表达式生成数字
var num = (10 * 10) - (5 * 5); // 75
```

### 8.5 条件混淆

```javascript
// 使用 && 或 || 代替 if
a && console.log(a);
b || console.error(b);
```

## 反混淆方法论

### 静态分析

1. **代码解析**: 使用 Babel/AST 解析代码结构
2. **模式识别**: 识别已知的混淆模式
3. **等价变换**: 将混淆代码替换为等价形式

### 动态分析

1. **运行时 Hook**: 拦截关键函数获取明文
2. **调试跟踪**: 追踪代码执行流程
3. **模拟执行**: 模拟执行解密逻辑

### 混合分析

1. **先静态后动态**: 静态分析提取关键函数，动态执行获取结果
2. **先动态后静态**: 动态分析获取密钥，静态解密还原
